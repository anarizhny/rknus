# RKNus — Chrome Extension

> Проверяет, находится ли текущий сайт в реестре заблокированных ресурсов Роскомнадзора.

## Возможности

- Проверка домена по локальной базе (~400K доменов) с O(1) lookup
- Badge на иконке: красный (заблокирован) / зелёный (чист)
- Popup с деталями блокировки (причина, дата, ID реестра)
- Автоматическая синхронизация базы (delta + full sync с fallback)
- Content script: баннер-предупреждение + подсветка заблокированных ссылок
- Bloom Filter для экономии памяти (176 KB вместо ~40 MB)
- Страница настроек (источник данных, частота синхронизации, тема, уведомления)
- Dark theme (system / manual)
- Локализация: русский + английский

## Установка (режим разработчика)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/anarizhny/rknus.git
   cd rknus
   npm install
   ```

2. Соберите расширение:
   ```bash
   npm run build
   ```

3. Откройте `chrome://extensions/` в Chrome

4. Включите **"Режим разработчика"** (переключатель в правом верхнем углу)

5. Нажмите **"Загрузить распакованное расширение"**

6. Выберите папку `dist/`

7. Расширение появится в панели расширений с иконкой "A"

## Privacy Policy

RKNus **не собирает, не хранит и не передаёт** персональные данные пользователей.

### Что делает расширение

- Скачивает публичный список заблокированных доменов из открытых источников (GitHub, API rublacklist)
- Сохраняет этот список **локально** на устройстве пользователя в IndexedDB
- При навигации проверяет домен текущей вкладки **по локальной базе** — без сетевых запросов

### Чего расширение НЕ делает

- **Не отправляет** посещаемые URL, домены или любую историю просмотров на внешние серверы
- **Не содержит** аналитики, телеметрии, трекеров или рекламы
- **Не собирает** данные о поведении пользователя
- **Не передаёт** данные третьим лицам

### Сетевые запросы

Расширение обращается к сети **исключительно** для скачивания базы заблокированных доменов:

| URL | Назначение | Когда |
|-----|-----------|-------|
| `https://raw.githubusercontent.com/zapret-info/z-i/master/dump.csv` | Полный дамп реестра (CSV) | Первый запуск, полная синхронизация |
| `https://reestr.rublacklist.net/api/v3/domains/` | Обновления реестра (JSON) | Delta-синхронизация каждые 1-24ч |

Никакие пользовательские данные (URL, домены, cookies, данные форм) не включаются в эти запросы.

### Хранение данных

| Хранилище | Содержимое |
|-----------|-----------|
| IndexedDB (`rknus`) | База заблокированных доменов (публичные данные из реестра РКН) |
| `chrome.storage.sync` | Настройки пользователя (тема, интервал синхронизации, вкл/выкл уведомлений) |
| `chrome.storage.local` | Timestamp последней синхронизации, сериализованный Bloom Filter |
| `sessionStorage` | Флаг "баннер закрыт" (только для content script, сбрасывается при закрытии вкладки) |

Все данные хранятся **локально** и удаляются при удалении расширения.

## Обоснование разрешений

Расширение запрашивает минимальный набор permissions, необходимый для работы:

| Permission | Зачем нужно |
|------------|------------|
| `activeTab` | Получить URL текущей вкладки, чтобы извлечь домен и проверить его по базе. Без этого разрешения расширение не может узнать, на каком сайте находится пользователь. |
| `tabs` | Слушать события навигации (`tabs.onUpdated`, `tabs.onActivated`), чтобы автоматически обновлять badge при переходе между вкладками и загрузке страниц. |
| `storage` | Сохранять настройки пользователя (`storage.sync`) и метаданные синхронизации (`storage.local`). Без этого настройки сбрасывались бы при каждом перезапуске браузера. |
| `alarms` | Создавать периодические таймеры для автоматической синхронизации базы доменов. Service Worker Chrome расширений не может использовать `setInterval` — только `chrome.alarms`. |
| `notifications` | Показывать системное уведомление при заходе на заблокированный сайт (отключается в настройках). |
| `scripting` | Регистрировать content script для отображения баннера-предупреждения на заблокированных сайтах и подсветки заблокированных ссылок (отключается в настройках). |

### Host permissions

| Host | Зачем нужно |
|------|------------|
| `https://reestr.rublacklist.net/*` | Скачивание обновлений базы заблокированных доменов через API. Расширение отправляет только GET-запросы к публичному API, без каких-либо пользовательских данных. |
| `https://raw.githubusercontent.com/*` | Скачивание полного CSV-дампа реестра из открытого GitHub-репозитория zapret-info/z-i. Используется как fallback, если основной API недоступен. |

### Content Scripts

Расширение загружает content script на все страницы (`<all_urls>`) для двух целей:
1. **Баннер-предупреждение** — показывает уведомление вверху страницы, если сайт заблокирован
2. **Подсветка ссылок** — выделяет на странице ссылки, ведущие на заблокированные домены

Content script можно полностью отключить в настройках расширения. Даже когда он активен, он **не читает** содержимое страницы и **не отправляет** никаких данных — только проверяет домены ссылок по локальной базе.
